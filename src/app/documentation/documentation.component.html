Documentation

This API follows the REST API principles and returns data in JSON format. Since a client ID and an api key have to be sent in the body as arguments along each request, the only supported HTTP verb is POST:

curl -X POST -d "clientID=yourClientID&api_key=yourApiKey" https://historical-car-api.web.app

Response

HTTP Status Code Definition Reason
200 OK Successful request
400 Bad Request Error on client side. Invalid client ID or quota of queries exhausted
401 Unauthorized Invalid api key
404 Not found Endpoint does not exist or client ID not found

Response might be cached to increase response time.
