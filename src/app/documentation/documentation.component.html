Documentation

This API follows the REST API principles and returns data in JSON format. Since a client ID and an api key have to be sent in the body as arguments along each request, the only supported HTTP verb is POST:

curl -X POST -d "clientID=yourClientID&api_key=yourApiKey" https://historical-car-api.web.app

Response

HTTP Status Code Definition Reason
200 OK Successful request
400 Bad Request Error on client side. Invalid client ID or quota of queries exhausted
401 Unauthorized Invalid api key
404 Not found Endpoint does not exist or client ID not found

Response might be cached to increase response time.

Example

const  baseURL = 'https://historical-car-api.web.app/cars?order=asc'

const  body = {
  method: 'POST',
  headers: {  'Content-Type': 'application/json'  },
  body: JSON.stringify({  'clientID': 'yourClientID', 'api_key': 'yourApiKey'  })

const  request = await fetch(baseURL, body)

const  response = await request.json()

console.log(response)

The key headers with the value application/json is necessary to let the server knows he will receive JSON.

This API is deployed on Google Cloud which optimises ressources on the server. Therefore, if not used during a certain amount of time, first API call may lead to a 'cold start', i.e. a slow start, which means that it might take a certain time. It then goes faster with next calls.
